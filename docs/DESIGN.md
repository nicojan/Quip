# Quip — design spec

**Quip** is a native macOS menu-bar app for finding a GIF and dropping it into
whatever you're typing in. It is a clean-room reimplementation of the idea
behind InaGif — none of InaGif's code, assets, or name carry over — built as a
sibling to Chorus (`~/dev/Chorus`).

> **Current status (shipping 1.1.6).** This is the original v1 spec; a few
> decisions below have moved on. The menu-bar surface uses the AppKit
> `NSStatusItem` + `NSPopover` fallback (driven by `AppDelegate`), not
> `MenuBarExtra`, so the global hotkey can open it. Trending, autocomplete,
> stickers, a content rating, drag-out, favorite filtering, a global hotkey, and
> gentle Sparkle update reminders shipped in v1.1. Since then: an inactivity
> reset back to the home view, a masked (reveal-on-tap) API-key field, launch-at-
> login error handling, VoiceOver support for the grid and controls, and a batch
> of copy/reliability fixes (see the 1.1.1–1.1.5 entries in `CHANGELOG.md`).
> **1.1.6 adds Collections** — named buckets (tags) for favourites, with a chip
> filter row on the home view and right-click filing on any GIF cell. Its design,
> code map, and a fine-tuning backlog live in
> `docs/superpowers/specs/2026-07-19-favourite-collections-design.md`. The
> "Open items" at the bottom are resolved. `docs/PLAN-v1.1.md` holds the v1.1
> plan; `release/DISTRIBUTION.md` is the live release runbook.

## Why a rewrite

The InaGif fork was a derivative of theramon's proprietary-licensed code and
could not be published. Quip is written from scratch, so it is ours to license
and ship however we like. The *idea* (a menu-bar GIF search that copies to the
clipboard, with favorites and recents) is not protectable; the *code and assets*
are new.

## Identity

- **Name:** Quip — a witty remark; a reaction GIF is a visual quip. One
  evocative word, in the spirit of "Chorus."
- **Bundle ID:** `com.nicojan.Quip`. Team `3CY4DX3K45`.
- **Icon:** same DNA as Chorus — a macOS squircle with a diagonal gradient and a
  single frosted-white glyph. Quip's gradient is **violet → pink**
  (`#7C3AED → #DB2777`) to distinguish it from Chorus's blue → violet. The glyph
  is a translucent-white **speech bubble with a play triangle** inside: "an
  animated reply." No text in the icon.
- **Palette (Chorus family, Tailwind tones):** near-black slate surface
  (`#0B0F1A`-ish), slate text (`#F1F5F9` / `#94A3B8`), one accent — violet
  `#7C3AED` for focus rings, selection, and the favorite star; pink `#DB2777`
  as the gradient partner. System font (SF).

## Platform & architecture

- SwiftUI, macOS 14.6+. **Menu-bar-only agent** (`LSUIElement = true`, no Dock
  icon).
- **Project generated by XcodeGen from `project.yml`** — same as Chorus. The
  `.xcodeproj` is a build artifact; `project.yml` is the source of truth. After
  changing settings/files, `xcodegen generate`.
- **Not sandboxed; Hardened Runtime ON** — matches Chorus and is what Sparkle
  auto-update + Developer-ID/notarized direct distribution needs. (Outside the
  sandbox, network and clipboard need no entitlement; the entitlements file is
  minimal.)
- Menu-bar surface via `MenuBarExtra("Quip", ...) { MenuContentView() }`
  with `.menuBarExtraStyle(.window)`. Modern and low-plumbing.
  - **Risk / fallback:** the `.window` style has limited control over popover
    sizing. Quip needs two sizes (narrow ~320×600, wide ~640×420). Verify the
    resize works early; if it fights us, fall back to `NSStatusItem` +
    `NSPopover` driven by an `AppDelegate` (still our own code).
- **Start at login:** `SMAppService.mainApp` (ServiceManagement) directly — no
  third-party dependency (InaGif used the LaunchAtLogin package; we drop it).
- **Auto-update:** **Sparkle** (SPM, 2.x), wired exactly like Chorus — a
  `SPUStandardUpdaterController` owned by `QuipApp`, all gated on
  `#if canImport(Sparkle)`, with a `CheckForUpdatesView`/`…ViewModel` pair. Since
  Quip has no main menu bar (agent app), the "Check for Updates…" button lives in
  **Settings** and the popover gear menu; background checks run on a daily
  schedule via Info.plist keys.
- **Dependencies (SPM):** **Sparkle** (auto-update) and **SDWebImageSwiftUI**
  (animated GIF thumbnails). Both open-source.
- **Small, focused files** (per house style): `QuipApp`, `MenuContentView`,
  `SearchBar`, `LayoutToggle`, `RecentSearchesRow`, `ResultsGrid`,
  `GifThumbnail`, `LibraryView`, `SettingsView`, `SearchViewModel`,
  `GiphyClient`, `GifLibrary`, `Theme` (color/spacing tokens), plus `Updater`
  (the Sparkle command, gated).

## Features (v1)

Carried over in spirit, all rebuilt:

- **Search** GIFs via the Giphy search API; **copy** a result to the clipboard
  as an animated file (write GIF bytes to a temp file, put the file URL on
  `NSPasteboard`).
- **Bring-your-own Giphy API key**, entered in Settings, stored in
  `UserDefaults`. No key bundled. Searching with no key shows an inline prompt
  that opens Settings.
- **Favorites** (star a GIF) and **Recently copied** (auto, de-duped, capped at
  24) — persisted as JSON in `UserDefaults` via `GifLibrary`.
- **Recent searches** — last 5, click to re-run.
- **Layout toggle** — 2-up (narrow) vs 5-up (wide), persisted.
- **Start at login**, toggled in Settings.

Giphy request: `GET https://api.giphy.com/v1/gifs/search?api_key=…&q=…&limit=36&rating=pg-13`.
Results parsed from `data[]`, using the `fixed_width` rendition for thumbnails.

**Stretch (not v1):** show Giphy *trending* GIFs when the popover opens with an
empty library and no search.

## UI redesign (improving on InaGif)

InaGif's current UI (for reference): a plain "InaGif" text button as the title,
a full-width "Toggle Layout" button, a search field plus a separate "Search"
button, recent-search chips that truncate to "c…", a big green "GIF copied!"
message, and a Preferences pane of bare checkboxes with a floating "Switch to
Main Window" button.

Quip's screens:

**Main popover**
- **Header bar:** small Quip glyph + "Quip" wordmark on the left; a **gear**
  button (opens Settings) on the right. Replaces the plain title button.
- **Search:** a single rounded field with an inline magnifier, focused when the
  popover opens. Searches on Return and on a short debounce as you type. The
  separate "Search" button is removed.
- **Layout toggle:** a small **2-up / 5-up segmented icon control** in the
  header. Replaces the full-width button.
- **Recent searches:** a **horizontally scrollable row of full chips** (no
  truncation), click to re-run.
- **Results grid:** rounded thumbnails, even spacing. **Hover** reveals a
  translucent overlay with a **star** (favorite toggle) and a copy hint.
  Clicking copies; confirmation is a subtle **checkmark toast**, not a green
  text block.
- **Default state (no active search):** the library — **Favorites** then
  **Recently copied**, each under a quiet section header. Empty favorites shows
  a one-line hint. No key set shows the Settings prompt.
- **Footer:** "Powered by GIPHY" — required by Giphy's terms — kept small and
  quiet.

**Settings** (a real settings surface, not floating checkboxes)
- **Giphy API Key** field + "Get a free key ↗" link + a live validity check
  (dims/greens based on a test call or last-request status).
- **Start at login** toggle.
- **Default layout** (narrow / wide).
- **Clear favorites** and **Clear recents** buttons.

## Distribution (mirrors Chorus's `release/DISTRIBUTION.md`)

Direct distribution: a Developer-ID-signed, **notarized `.dmg`**, with Sparkle
handling in-app updates. Not sandboxed; Hardened Runtime on. Hosted from a public
`nicojan/Quip` GitHub repo.

- **Appcast:** `docs/appcast.xml`, served by GitHub Pages (`main` → `/docs`) at
  `https://nicojan.github.io/Quip/appcast.xml` — this is `SUFeedURL`.
- **DMGs:** attached as GitHub Release assets, one per version (tag `vX.Y.Z`);
  the appcast `<enclosure>` points at the release download URL.
- **Signing key:** reuse the existing EdDSA key already in the login Keychain
  (shared with WatchMeType and Chorus), so `SUPublicEDKey` =
  `6/h2Pfjbo39vHie8JIt/kY7h0wQvmQxj9Ea0W3gnH0w=`. No new key to generate.
- **Team:** `3CY4DX3K45`, via `release/ExportOptions.plist` (method
  `developer-id`).

Quip ships its own `release/DISTRIBUTION.md` adapting Chorus's step-by-step
(archive → export → create-dmg → notarytool → staple → gh release → sign_update →
appcast item → commit). Actually cutting a release needs the Apple account +
Keychain + the public GitHub repo, so it happens when you're ready, not in the
first build.

## Repo layout & docs (Chorus conventions)

```
Quip/
  project.yml                 # XcodeGen source of truth
  Quip/                       # app sources + Info.plist + Quip.entitlements + Assets
  QuipTests/                  # GifLibrary + GiphyClient tests
  docs/DESIGN.md              # this spec
  docs/appcast.xml            # Sparkle feed (GitHub Pages)
  release/DISTRIBUTION.md     # release runbook (adapted from Chorus)
  release/ExportOptions.plist
  scripts/make_icon.py        # regenerates the AppIcon PNG set
  README.md  CHANGELOG.md  LICENSE
```

- **README / CHANGELOG / any public copy** follow Chorus's rule: plain, direct,
  active prose (Orwell's six rules) and pass the `mcp-humanizer` check
  (check → fix → re-check until clear). MIT license (ours to choose now).

## Verification

- Builds via `xcodebuild -scheme Quip -destination 'platform=macOS'`.
- Unit tests for `GifLibrary` (favorites/recents/persistence) and `GiphyClient`
  URL building + response parsing.
- Manual smoke test: no-key prompt → enter key → search → copy → paste; favorite
  and recents populate; layout toggle resizes; start-at-login registers.
